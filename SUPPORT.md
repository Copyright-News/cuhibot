# Support

We want to make sure you have a great experience using **Cuhi Bot**. If you run into issues or have questions, here is how you can get help.

## 📖 1. Read the Documentation

Before reaching out, please check the following resources—chances are your question has already been answered:

*   **[README](../README.md)**: Setup, architecture, and configuration basics.
*   **[CONTRIBUTING](../CONTRIBUTING.md)**: Guidelines for local development and PR submission.
*   **[SECURITY](../SECURITY.md)**: Security practices and reporting instructions.

## 🐛 2. Found a Bug?

If you have found a bug in the bot's logic, a broken download platform, or an issue with the Mini App UI:

1.  **Search the Issue Tracker:** Check if someone else has already reported it in our [GitHub Issues](https://github.com/copyrightnews/cuhibot/issues).
2.  **Open a New Issue:** If it's a new bug, please open an issue using our Bug Report template. Include as much detail as possible, including logs, your environment, and steps to reproduce.

*Note: For security vulnerabilities, do not use the public issue tracker. See [SECURITY.md](../SECURITY.md) for private reporting instructions.*

## 💡 3. Feature Requests

Have an idea to make Cuhi Bot better? We'd love to hear it!
Open a new issue on GitHub and apply the `enhancement` label. Describe your use case and how the feature would work.

## 💬 4. Community & Direct Help

If you need help setting up your self-hosted instance, have questions about cookie extraction, or just want to chat with the maintainers:

*   **Telegram Channel:** Join our official Telegram community at [@copyrightnews](https://t.me/copyrightnews) for updates, announcements, and peer-to-peer support.
*   **Email Support:** For private inquiries or business-related questions, you can reach out to the lead maintainers directly at `mintdmca@gmail.com`.

We are a small team of open-source enthusiasts, and we try our best to respond to all inquiries within 48 hours. Thank you for using Cuhi Bot!
